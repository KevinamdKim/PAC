# PAC 代理配置文件

## 项目说明

这是一个 PAC（代理自动配置）文件集合，用于浏览器和客户端的智能代理配置。通过该文件，网络请求会根据不同的目标域名自动选择合适的代理方案。

## 文件说明

- `OmegaProfile_gfw_.pac` - 网站代理配置文件，包含针对不同网站的代理策略

## 功能特性

该 PAC 文件包含以下配置规则：

- **技术开发资源**：arXiv、GitHub、Docker 等开发资源配置
- **云服务**：各类云服务和开发工具配置

## 使用方法

### 浏览器配置

1. 打开浏览器的代理设置
2. 选择 "使用 PAC 脚本"
3. 输入本文件的访问路径或本地路径，例如：
   - 本地：`file:///path/to/OmegaProfile_gfw_.pac`
   - 远程：`http://your-server/OmegaProfile_gfw_.pac`

### 常见浏览器

- **Chrome/Chromium**：设置 → 高级 → 系统 → 代理设置
- **Firefox**：首选项 → 网络 → 连接 → 自动配置 PAC URL
- **Edge**：设置 → 隐私和安全 → 代理

## 工作原理

PAC 脚本通过 JavaScript 执行，根据目标 URL 的主机名进行正则表达式匹配，返回相应的代理类型：

- `"+本地代理接入"` - 走代理
- `"DIRECT"` - 直接连接

## 注意事项

- 确保代理服务正常运行
- 某些应用可能不支持 PAC 配置，需要手动设置代理
- 定期更新文件以支持新的网站规则
